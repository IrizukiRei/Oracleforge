# 占い画面（Reading Screen）UI／オーバーレイ設計指示書

## 前提条件

- ルノルマンカード画像はプリセット固定（ユーザー編集不可）
- 画像は WebP ファイルパス参照（`card-images/*.webp`）、起動時にプリロード済み
- データはすべて個人端末ローカル保存（IndexedDB / localStorage）
- 背景デザインは現状維持。背景を変えずに、カード・テキスト・オーバーレイの視認性を保つ構造のみを定義する

---

## 1. レイヤー構造

占い画面を **5つのレイヤー** に分離する。下から順に積む。

```
z-index
  ─────────────────────────────────────────────────

  9999   [L5] モーダル層
          用途：画像拡大、確認ダイアログ、エラー表示
          常に最前面。他の全レイヤーを覆う。

  ─────────────────────────────────────────────────

  500    [L4] 広告・課金・ガイド拡張層（将来用）
          用途：インタースティシャル広告、課金導線、
               チュートリアルバルーン、お知らせバナー
          現時点では空。後から差し込む枠だけ確保する。

  ─────────────────────────────────────────────────

  100    [L3] オーバーレイ層
          用途：演出背景（overlay.webp）、状態テキスト、
               操作ロック
          展開〜結果確定の演出を一手に担う。

  ─────────────────────────────────────────────────

  10-50  [L2] インタラクション層
          用途：操作ボタン（シャッフル/スキップ/リセット）、
               ヒントテキスト、「まとめて集める」ボタン、
               補足カードボタン
          ユーザーが直接触れる要素すべて。

  ─────────────────────────────────────────────────

  1-9    [L1] カード表示層
          用途：スプレッド枠（card-slot）、カード本体（card-inner）、
               シャッフルアニメーション用カード山
          カードの配置・反転・表示のみ。操作判定は持たない。

  ─────────────────────────────────────────────────

  0      [L0] 背景層（変更しない）
          既存の背景グラデーション＋spread-display枠線。
```

### 現状の問題

現在の実装では上記の役割分離がない。具体的には：

- **ボタン（L2）とカード（L1）が同じ階層に混在**している。`.controls` が `spread-display` と同列にあるだけで、z-index による明示的な分離がない
- **オーバーレイ（L3）が `document.body` に直接追加**されており、reading-screen のスコープ外に漏れている
- **操作ロックが存在しない**。シャッフルアニメーション中もボタンがクリック可能で、多重実行のリスクがある
- **ヒントテキスト（「カードをタップして集めてください」）がシャッフル関数内でインラインHTML生成**されており、L2としての管理ができていない

## 2. 各レイヤーの詳細

### L1: カード表示層

**役割**: カードの「見た目」だけを担当する。タップ判定やボタンは置かない。

| 要素 | 説明 |
|------|------|
| `.card-slot` | スプレッド上の定位置枠。空のときは半透明の破線枠 |
| `.card-inner` | カード本体。裏面 (`.card-back`) と表面 (`.card-front`) を内包 |
| `.shuffle-card` | シャッフル演出時の仮カード山（12枚）。演出終了後に破棄 |
| `.spread-inner` | 全 `.card-slot` を内包するコンテナ |

**制約**:
- カード表面の `onclick`（画像拡大）は L5 モーダルへの橋渡しのみ行う
- カードの配置座標は `getSpreadPositions()` が返す `{x, y, label}` で決まる
- シャッフルカード山（`.shuffle-card`）は演出専用。L1 の一時的な住人として扱い、回収完了後に即座にDOMから除去する

### L2: インタラクション層

**役割**: ユーザーが触れるすべてのUI部品を集約する。

| 要素 | z-index | 表示条件 |
|------|---------|----------|
| シャッフルボタン | 50 | 初期状態・リセット後 |
| スキップボタン | 50 | 初期状態・リセット後 |
| リセットボタン | 50 | 常時（ただしアニメ中は disabled） |
| ヒントテキスト | 50 | シャッフル散開後〜回収完了まで |
| 「まとめて集める」ボタン | 50 | シャッフル散開後〜回収完了まで |
| 補足カードボタン | 50 | 結果表示後、上限未達のとき |

**制約**:
- ヒントテキストは固定HTMLとして `reading-screen` 内に事前配置し、`display:none` で隠す。シャッフル関数内でのインライン生成をやめる
- 「まとめて集める」ボタンも同様に事前配置する
- ボタンの `disabled` 状態は L3 オーバーレイの状態と連動させる（後述の状態遷移を参照）

### L3: オーバーレイ層

**役割**: 画面全体を覆う演出幕＋操作ロック。

**実装方針**:
- `reading-screen` の **直下の子要素** として固定配置する（現状の `document.body` 追加をやめる）
- `position: absolute; inset: 0;` で reading-screen 内を覆う
- `pointer-events: auto` で、オーバーレイが出ている間は下層への操作を遮断する

**視認性の確保**:
- `overlay.webp`（背景画像）は `opacity: 0.7` → カード面は見えるが文字は読まない想定
- テキスト誘導が必要な場面（回収中）ではオーバーレイの opacity を `0.4` まで下げるか、テキストを L3 内（オーバーレイの上）に配置して白文字+ドロップシャドウで視認性を担保する
- 結果確定時のフェードアウトは `transition: opacity 2s ease-out` で自然に消す

### L4: 拡張層（現時点では空）

**役割**: 将来の課金・広告・ガイドUIのための予約領域。

現在は何も置かない。このレイヤーの存在だけを設計上確保しておく。
具体的な配置案は本書末尾の「拡張ポイント」節で述べる。

### L5: モーダル層

**役割**: カード画像の拡大表示、汎用確認ダイアログ。

- 既存の `#imageModal`、`#genericModal` がここに属する
- 現状の `z-index: 9999〜10000` をそのまま維持
- 変更不要

## 3. 状態遷移

占い画面は以下の **5つの状態** を持つ。

```
[待機] → [シャッフル中] → [回収中] → [展開中] → [結果確定]
  ↑                                                  │
  └──────────── リセット ←───────────────────────────┘
```

### 各状態の定義

#### (A) 待機（idle）

- **画面**: 空のカード枠（破線）のみ表示
- **L2 ボタン状態**:
  - シャッフル: **有効**
  - スキップ: **有効**
  - リセット: **無効**（何も置かれていないため不要）
- **L3 オーバーレイ**: 非表示
- **操作ロック**: なし
- **遷移先**: シャッフルボタン押下 → (B)、スキップボタン押下 → (D)即時

#### (B) シャッフル中（shuffling）

- **画面**: カード山（12枚）がリフル → 散開アニメーション
- **L2 ボタン状態**:
  - シャッフル: **無効（disabled）**
  - スキップ: **無効（disabled）**
  - リセット: **有効**（中断手段として残す）
- **L3 オーバーレイ**: 非表示
- **操作ロック**: ボタンを disabled にするだけで十分。カード山のタップは次の状態への遷移トリガーなのでロックしない
- **所要時間**: リフル 200ms → 散開 1500ms → 合計約 1.7s
- **遷移先**: 散開完了 → (C) に自動遷移

#### (C) 回収中（collecting）

- **画面**: 散開したカードが画面上に散らばっている。ヒントテキストと「まとめて集める」ボタンを表示
- **L2 ボタン状態**:
  - シャッフル: **無効**
  - スキップ: **無効**
  - リセット: **有効**
- **L2 テキスト**: 「カードをタップして集めてください」を表示
- **L3 オーバーレイ**: 非表示
- **操作ロック**: なし（カードタップが主操作）
- **遷移先**: 必要枚数分を回収完了 or「まとめて集める」押下 → (D) へ

#### (D) 展開中（revealing）

- **画面**: スプレッド枠にカードが順次配置され、1枚ずつ裏→表に反転
- **L2 ボタン状態**:
  - シャッフル: **無効**
  - スキップ: **無効**
  - リセット: **無効**（展開中の中断は体験を損なうため不可）
- **L3 オーバーレイ**: **表示（opacity: 0.7、pointer-events: auto）**
  - `overlay.webp` を背景に、画面全体を覆う
  - `pointer-events: auto` で L1・L2 への一切の操作を遮断
  - カード反転アニメーション（0.6s × 枚数、400ms間隔）の間ずっと表示
- **操作ロック**: **オーバーレイによる完全ロック**
- **所要時間**: 3000ms（初期待機）+ 枚数 × 400ms
- **遷移先**: 全カード反転完了 → (E) へ

#### (E) 結果確定（result）

- **画面**: 全カードが表向きで配置済み。結果テキスト（カード名・意味・説明）を下部に表示
- **L2 ボタン状態**:
  - シャッフル: **無効**
  - スキップ: **無効**
  - リセット: **有効**
  - 補足カードボタン: **有効**（上限3枚未満のとき）
- **L3 オーバーレイ**: **フェードアウト → 非表示**
  - `opacity: 0.7 → 0` を 2s かけて遷移
  - 遷移完了後に DOM から除去
  - フェードアウト中は `pointer-events: none` に切り替え、結果テキストの閲覧を即座に許可する
- **操作ロック**: なし
- **広告フラグ**: `hasSpreadResult = true` をセット。この画面を離れたとき（showScreen 呼び出し時）にインタースティシャル広告を表示
- **遷移先**: リセット押下 → (A) へ

### スキップ時の特殊フロー

```
[待機] → スキップ押下 → [結果確定]
```

- (B)(C)(D) をすべてスキップし、即座に全カードを表向きで配置
- オーバーレイは表示しない
- SE（効果音）も鳴らさない
- 演出なしの高速フロー

## 4. オーバーレイ詳細仕様

### 表示タイミングと挙動

| 状態 | オーバーレイ | opacity | pointer-events | 背景画像 |
|------|-------------|---------|----------------|----------|
| 待機 | 非表示 | - | - | - |
| シャッフル中 | 非表示 | - | - | - |
| 回収中 | 非表示 | - | - | - |
| 展開中 | **表示** | 0.7 | **auto（ロック）** | overlay.webp |
| 結果確定（遷移中） | フェード中 | 0.7→0 | **none（解除）** | overlay.webp |
| 結果確定（完了後） | 非表示 | - | - | - |

### 現状からの変更点

1. **オーバーレイの配置先**: `document.body` → `#reading-screen` 直下
   - 理由: reading-screen のスコープ内に閉じ込めることで、他の画面に影響しない
2. **pointer-events の付与**: 現状は `none` 固定 → 展開中は `auto` にして操作を完全遮断
3. **フェードアウト開始タイミング**: 現状は `showResults()` 内で開始 → 最後のカード反転完了をトリガーにする（タイミングの根拠を明確化）

## 5. 視認性ガイドライン

背景デザインは変更しないため、各要素の視認性は以下のルールで担保する。

### カード（L1）

- カード画像は `object-fit: cover` でスロット全体に表示
- カード枠線: `2px solid rgba(100, 200, 255, 0.8)`（配置後）
- シグニフィケーター: `box-shadow: 0 0 15px rgba(255, 200, 100, 0.8)` で金色グロウ
- **背景との分離**: カード自体が不透明画像なので、背景に埋もれることはない

### テキスト（L2）

- ヒントテキスト: `color: #fff` + `text-shadow: 0 0 8px rgba(0,0,0,0.8)` で暗背景でも読める
- ボタンラベル: 現状の `rgba(100,200,255)` 系を維持。背景の `rgba(0,0,0,0.3)` パネルとの組み合わせで十分な対比
- 結果テキスト: `.result-info` の `background: rgba(255,255,255,0.1)` は背景が暗い前提で成立する。背景が明るい場合は破綻するが、背景を変えない前提なので問題なし

### オーバーレイ（L3）

- `overlay.webp` は暗色寄りの画像 + `opacity: 0.7` で背景を適度に隠す
- カード表面は L1 にあり、オーバーレイの下に位置する。展開中にカードが見えることが重要なので、`opacity` は `0.7` を上限とする（これ以上暗くするとカードが見えなくなる）
- テキストをオーバーレイの上に出す場合は L3 内の子要素として配置し、白文字 + ドロップシャドウで対比を確保する

## 6. 拡張ポイント（課金・広告・ガイドを足すならここ）

### 広告

**現状**: インタースティシャル広告は「占い画面を離れるとき」に `showScreen()` 内で発火。reading 画面の中には広告UIがない。

**追加案**:

- **L4 にバナー広告枠を設置**: `reading-screen` 下部（結果テキストの下）に高さ 50px の固定バナー枠を確保する。`position: sticky; bottom: 0;` でスクロールしても常時表示。課金ユーザーには `display: none`
- **リワード広告（補足カード）**: 補足カードの上限3枚を超えて引きたい場合、リワード広告視聴で +1枚を開放する。ボタンは L2 の補足カードボタンの隣に配置。`[広告を見て追加カードを引く]`
- **インタースティシャルのタイミング**: 現状の「画面離脱時」に加え、リセット→再シャッフル時（同じ画面内での再占い）にも発火条件を追加できる。L4 にインタースティシャル用の透明トリガー層を置く設計にしておけば、条件分岐だけで対応可能

### 課金（IAP）

**現状**: `IAPManager` は存在するが、reading フロー内に課金導線がない。

**追加案**:

- **広告除去**: L4 のバナー枠の表示/非表示を `iapManager.adsDisabled` フラグで制御するだけ
- **プレミアムスプレッド**: スプレッド選択画面（`spread-screen`）で一部スプレッドに鍵アイコンを付け、タップ時に課金モーダル（L5）を表示。reading 画面自体の変更は不要
- **スプレッド保存/履歴**: 結果確定後に「この結果を保存」ボタンを L2 に追加。無課金では直近3件まで、課金で無制限。保存先は IndexedDB

### ガイド／チュートリアル

**追加案**:

- **初回ガイド**: 初めて reading 画面を開いたとき、L4 に半透明マスク＋吹き出しを表示する。ステップ:
  1. 「シャッフルボタンを押してください」→ シャッフルボタンをハイライト
  2. 「カードをタップして集めてください」→ 散開カードをハイライト
  3. 「結果が表示されます」→ 結果エリアをハイライト
- **実装方法**: L4 に `position: absolute; inset: 0; background: rgba(0,0,0,0.6)` のマスクを敷き、ハイライト対象の座標に `clip-path` で穴を開ける。吹き出しは absolute 配置の `<div>`
- **発火条件**: `localStorage.getItem('tutorialCompleted')` が `null` のとき。完了後にフラグを立てる
- **スキップ**: 「スキップ」ボタンを吹き出し内に常時表示。途中離脱可能

### L4 の具体的なHTML構造案

```html
<!-- reading-screen 内、L3オーバーレイの直後に配置 -->
<div id="extension-layer" style="
    position: absolute; inset: 0;
    z-index: 500;
    pointer-events: none;
">
    <!-- 広告バナー（課金時は非表示） -->
    <div id="ad-banner-slot" style="
        position: absolute; bottom: 0; left: 0; right: 0;
        height: 50px;
        pointer-events: auto;
        display: none;
    "></div>

    <!-- チュートリアル（初回のみ） -->
    <div id="tutorial-overlay" style="display: none;"></div>

    <!-- お知らせバナー -->
    <div id="notice-banner" style="display: none;"></div>
</div>
```

`pointer-events: none` をレイヤー全体に設定し、中の個別要素だけ `auto` にすることで、通常時は下層の操作を邪魔しない。

## 7. 現状コードとの差分まとめ

この指示書の設計を実装する場合、以下の変更が必要になる。

| 項目 | 現状 | 変更後 |
|------|------|--------|
| オーバーレイの親要素 | `document.body` | `#reading-screen` 直下 |
| オーバーレイの pointer-events | 常に `none` | 展開中 `auto`、結果確定で `none` → 除去 |
| ボタン disabled 制御 | なし | 状態遷移に連動して切り替え |
| ヒントテキスト生成 | `shuffleAndDraw()` 内でインライン生成 | HTML に事前配置、表示/非表示を切り替え |
| 「まとめて集める」ボタン | `shuffleAndDraw()` 内でインライン生成 | HTML に事前配置 |
| 状態管理変数 | なし（暗黙的） | `readingState` 変数（'idle' / 'shuffling' / 'collecting' / 'revealing' / 'result'）を明示的に追加 |
| z-index 体系 | バラバラ（2, 20-31, 50, 100, 120, 200, 9999） | 5層に整理（L1:1-9, L2:10-50, L3:100, L4:500, L5:9999） |
| L4 拡張層 | 存在しない | 空の `#extension-layer` を追加 |
